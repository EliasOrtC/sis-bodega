All of the commands except `eject` will still work, but they will point to the copied scripts so you can tweak them. At this point you're on your own.

You don't have to ever use `eject`. The curated feature set is suitable for small and middle deployments, and you shouldn't feel obligated to use this feature. However we understand that this tool wouldn't be useful if you couldn't customize it when you are ready for it.

## El sistema de selección de filas funciona así:

Contexto Global (SelectionContext.jsx): Crea un contexto de React que almacena selectedItem (el objeto de la fila seleccionada, o null si ninguna) y setSelectedItem (función para actualizarlo). Esto permite compartir el estado entre componentes sin props drilling.

Proveedor en App.js: SelectionProvider envuelve toda la app, haciendo que selectedItem esté disponible globalmente.

Selección en Módulos: Cada módulo (Empleados, Clientes, etc.) usa useSelection() para acceder al contexto. En cada TableRow, onClick={() => setSelectedItem(item)} selecciona la fila, y selected={selectedItem?.id === item.id} resalta visualmente la fila con un fondo rojo claro.

Botón Editar en FloatingMenu: El menú flotante usa useSelection() y muestra el botón "Editar [Módulo]" solo si selectedItem existe y el módulo actual coincide. Al hacer click, abre un modal con los campos prellenados del selectedItem.

Deselección Automática: En App.js, un useEffect con location.pathname como dependencia ejecuta setSelectedItem(null) al cambiar de módulo. Otro useEffect con beforeunload deselecciona al cerrar la app.

Esto permite una selección única global que se resetea automáticamente al navegar, manteniendo la UI limpia y funcional.
